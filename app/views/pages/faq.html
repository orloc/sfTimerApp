What is the intent of this project?

To provide a integrated easy to use place to share timers and aggregate mob tracking efforts.
Originally - I made this for myself to be more competitive while hunting Seafuries, I hated always opening up many different timer windows and wanted something that I could share with a friend.

Since those days I have completely rewritten the small application this started out to be into a more robust fully featured tracking "foundation" upon which I plan to extend / add new features as the current feature set gets polished out and bugs get fixed.

How can this make my life better?

If you have ever wanted to keep / have kept timers in the past you know it can be frustrating.

  • Never have to worry about "setting up" timers again!
  • Save pre-configured timer groups for specific zones and share them with people you often group with.
  • Spread time keeping responsibility
  • Provide transparency to the raiding process for all

What does this do ?

You can create "timer groups" those timer groups can be thought of as buckets of timers. Each group can be shared with specific individuals based on the owner(s) choosing. At the moment there is only one type of permission that can be granted

  • Authorization / Login / Registration
  • Management of federated timer groups
  • Creation of timers inside timer groups
  • Real time updates for users looking at the same board.
  • Sharing of timer groups ( no distinction between roles at this time - all are owners)

What does not do?

Parse your client logs or interact with your physical machine / game. ( v2 maybe )

What is planned to be done?

  • Add better support for roles / permissions - implement read and write permissions on invitations
  • Add more "action cues" - who is viewing current group // who did what action on a group you are viewing // better visualization of actions
  • Integration tests across 3 primary components
  • Implement reset password functionality
  • Better cross browser test coverage
  • Implement persistent message store to allow for socket connection loss without user interruption
  • Implement server side timer 'audits' for running timers to counter client side cloak drift / various architecture clock speeds.

How do I use this application ?

Upon registration / login you will see something similar to the image below. The first thing you will want to do is create a new timer group found via the form in the bottom left hand corner.

The leftmost picture below has three section markers which will be used in later parts of the guide.

Account / Invitation Management ( top right )

Menu (top left)

Controls ( bottom left)

Once you have added a group you will see it appear underneath the MENU. Click on this group to select it. You will see the screen / controls change to a new section.

Buttons on the controls section are split into TWO GROUPS - "Timers" and "Group" control sections.

  • Timer Section
    • Adds a new timer
  • Group Section
    • Adds a new group
    • Manage permissions for the selected group
    • Edit's currently selected group's name / description
    • Deletes the currently selected group and all associated timers / initiations
